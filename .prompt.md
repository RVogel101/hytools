Title: Improve OCR for Western Armenian textbook

Purpose:
Help the developer improve the OCR pipeline used in `scripts/reprocess_textbook_ocr.py` so per-page text extraction is more complete (fewer blank pages, higher recall of Armenian text). Use the provided sample outputs in `data/textbook_ocr_improved/` to diagnose failures, propose reproducible experiments, and deliver small, testable code patches.

Available context (workspace paths):
- `scripts/reprocess_textbook_ocr.py` — current OCR runner to improve.
- `data/textbook_ocr_improved/` — per-page `.txt` outputs produced by the script (one file per page).
- Project OCR helpers under `hytools.ocr.*` (use repository search if you need exact module files).

Primary task (what the assistant should do):
- Analyze sample OCR outputs and identify common failure modes (blank outputs, single-digit outputs, truncated lines, wrong language selection).
- Propose a prioritized list of experiments to improve recall (e.g., increase DPI, try per-page language enforcement, change binarization, vary `psm`, run multiple passes with alternate configs).
- Produce minimal, self-contained code patches (apply_patch-ready diffs) to `scripts/reprocess_textbook_ocr.py` or OCR helpers.
- Provide runnable commands to reproduce experiments and a compact metric report (blank-page count, avg chars/page, pages with < N chars).

Inputs the assistant can request or assume:
- `pdf_path` (optional): path to the input PDF to re-run OCR on.
- `sample_dir`: directory with existing per-page `.txt` outputs (defaults to `data/textbook_ocr_improved/`).
- numeric ranges for experiments: `dpi` (200–600), `confidence_threshold` (0–100), list of `psm` values, `binarization` options.
- `lang_mode`: one of `hye`, `hye+eng`, `eng`, or `auto` (explicit Western Armenian preferred: `hye` or `hye+eng`).

Expected outputs from the assistant:
- Short diagnostic summary with 3–6 clear failure modes found in sample outputs.
- A ranked experiment plan (small matrix of config variants to try) and the exact commands to run.
- Up to 3 concrete code patches with short descriptions and automated metric checks to validate improvement.
- If applicable, a lightweight script or helper (one file) to compute metrics across `page_*.txt` and output a CSV/summary.

Preferred output style and constraints:
- Keep changes minimal and focused on correctness and reproducibility.
- Prefer small, reviewable patches over large rewrites.
- When suggesting new dependencies, prefer standard Python packages already used in this repository; explicitly state installation commands.

Example assistant invocation (use when acting):
Given `sample_dir=data/textbook_ocr_improved/`, analyze 20 sample pages and return:
1) 3 concrete failure modes and supporting examples (file names and snippets),
2) a 4-step experiment plan with exact commands, and
3) one small patch to `scripts/reprocess_textbook_ocr.py` that adds a `--debug-sample N` flag to rerun selected pages with alternate configs.

Ambiguities to clarify (ask the user):
- Do you prefer faster iterative experiments (low DPI) or higher-quality but slower runs (high DPI)?
- Should the assistant target only changes to `scripts/reprocess_textbook_ocr.py` or may it edit lower-level OCR helpers under `hytools.ocr`?
- Confirm preferred language variant: Western Armenian (`hye`) or mixed (`hye+eng`).

Iteration and next steps suggestions:
- After experiment results, produce a final patch that implements the best-performing settings as defaults and add a `--reprocess-missing` mode that only reruns pages below a char-count threshold.
- Optional: add unit tests for the text-analysis helper and a short README snippet describing typical experiment commands.

Notes for maintainers:
- Do not invent Armenian text examples—always use text from `data/` files when demonstrating failures or fixes.
- Preserve existing CLI flags and behavior unless a clear backward-compatible improvement is provided.
